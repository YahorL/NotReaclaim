# NotReclaim Persistence Layer — Design Spec (Milestone 2)

**Date:** 2026-05-28
**Status:** Approved (ready for implementation planning)
**Parent design:** `docs/superpowers/specs/2026-05-28-notreclaim-design.md`
**Depends on:** Milestone 1 — `@notreclaim/scheduler` (pure engine, merged)

## Summary

Milestone 2 builds the persistence layer for NotReclaim: a Postgres schema and
migrations for all core entities, repositories the later sync/API layers will
use, and pure mappers that convert DB rows into the scheduler engine's domain
types. All data is `userId`-scoped (single user now, multi-user-capable).

Database access uses **Prisma** with a real Postgres (system-installed service).
A separate test database backs repository integration tests.

## Goals

- A Prisma schema + migration for `User`, `Settings`, `CalendarEvent`, `Task`,
  `Habit`, `ScheduledBlock`, all `userId`-scoped with cascade deletes.
- Repositories with focused, `userId`-scoped CRUD + query methods.
- Pure, unit-tested mappers from DB rows to the engine's domain types (for the
  horizon-independent entities).
- Integration tests against a real test Postgres; pure unit tests for mappers.

## Non-Goals (Milestone 2)

- OAuth flow and token encryption (M3).
- Google Calendar sync, syncToken/watch-channel storage (M3).
- `Habit → engine Habit` expansion, working-hours → `workingWindows` expansion,
  and full `ScheduleInput` assembly (all M3 — they need horizon + timezone).
- Any HTTP/API surface or WebSocket (M4).

## Package

New workspace package **`packages/db`** (`@notreclaim/db`), depending on
`@notreclaim/scheduler` for the domain types its mappers target. It owns the
Prisma schema, the generated client, repositories, and mappers. Vitest for tests
(consistent with M1).

```
packages/db/
  package.json                # @notreclaim/db
  tsconfig.json
  vitest.config.ts
  prisma/
    schema.prisma
    migrations/               # generated by `prisma migrate`
  src/
    client.ts                 # shared PrismaClient instance
    config.ts                 # typed DATABASE_URL / TEST_DATABASE_URL access
    errors.ts                 # ConflictError, NotFoundError + Prisma error translation
    mappers.ts                # pure DB-row -> engine-domain-type functions
    repositories/
      user-repository.ts
      settings-repository.ts
      calendar-event-repository.ts
      task-repository.ts
      habit-repository.ts
      scheduled-block-repository.ts
    index.ts                  # public exports
  test/
    mappers.test.ts           # pure, no DB
    repositories/*.test.ts     # integration, real test Postgres
    setup.ts                  # global setup: migrate test DB; per-test truncate
```

## Prisma Setup

- One `schema.prisma` with a `postgresql` datasource reading `DATABASE_URL`.
- Migrations authored with `prisma migrate dev`; applied in test/prod with
  `prisma migrate deploy`.
- A single shared `PrismaClient` exported from `src/client.ts`.

## Storage Conventions

These keep DB rows round-tripping cleanly through the pure engine:

- **Timestamps** → `timestamptz` columns. The engine uses epoch milliseconds, so
  mappers convert `Date → date.getTime()` on the way out.
- **Durations** (`durationMs`, `minChunkMs`, `maxChunkMs`, `chunkMs`) → integer
  milliseconds, matching the engine 1:1 (no unit conversion).
- **IDs** → UUID strings (`@default(uuid())`) — exactly the `string` ids the
  engine expects.

## Schema (entities)

All entities have `id`, `createdAt`, `updatedAt`. All non-`User` entities have
`userId` with a cascade-delete relation to `User`.

### User
- `email` — text, unique.
- `googleId` — text, unique, nullable (set when OAuth lands in M3).
- `googleRefreshToken` — text, nullable (M3's auth layer encrypts the value).
- Relations: one `Settings`; many of each scoped entity.

### Settings (1:1 with User)
Consolidates all user preferences (resolves the parent spec's timezone /
working-hours overlap).
- `userId` — unique FK.
- `timezone` — IANA string (e.g. `America/New_York`).
- `workingHours` — JSONB: array of `{ weekday: 0–6, startMinute, endMinute }`.
- `horizonDays` — int, default 14.
- `defaultMinChunkMs` — int.
- `defaultMaxChunkMs` — int.

### CalendarEvent (cached fixed events)
- `googleCalendarId` — text.
- `googleEventId` — text.
- `title` — text.
- `startsAt`, `endsAt` — timestamptz.
- Unique: `(userId, googleCalendarId, googleEventId)`.

### Task
- `title` — text.
- `priority` — int (lower = scheduled first).
- `durationMs` — int.
- `dueBy` — timestamptz.
- `minChunkMs`, `maxChunkMs` — int.
- `category` — text, nullable.
- `status` — enum `TaskStatus { pending, scheduled, completed, archived }`,
  default `pending`.
- `timeLoggedMs` — int, default 0.

### Habit (recurrence rule, not concrete occurrences)
- `title` — text.
- `priority` — int.
- `chunkMs` — int (duration of one occurrence).
- `perPeriod` — int (occurrences per period).
- `periodType` — enum `HabitPeriod { week }` (extensible), default `week`.
- `preferredStartMinute`, `preferredEndMinute` — int, nullable
  (minutes from local midnight).
- `eligibleDays` — int array (weekdays 0–6).
- `status` — enum `HabitStatus { active, paused }`, default `active`.

### ScheduledBlock
- `title` — text.
- `startsAt`, `endsAt` — timestamptz.
- `pinned` — bool, default false.
- `googleEventId`, `googleCalendarId` — text, nullable (set when mirrored to
  Google in M3).
- `taskId` — uuid, nullable, FK → Task (cascade).
- `habitId` — uuid, nullable, FK → Habit (cascade).
- **Check constraint:** exactly one of `taskId`, `habitId` is set. The mapper
  derives `sourceType`/`sourceId` from whichever is present.

## Repositories

One module per entity; thin wrappers over the Prisma client. Every read scopes by
`userId`; every mutation verifies the row belongs to the user. M2 covers CRUD +
obvious queries. Sync-specific bulk operations (replace-horizon, syncToken
bookkeeping) are deferred to M3.

- **UserRepository** — `create`, `findById`, `findByEmail`, `findByGoogleId`,
  `update`.
- **SettingsRepository** — `getByUserId`, `upsert`.
- **CalendarEventRepository** — `listByUserInRange(userId, start, end)`,
  `upsertMany`, `deleteByGoogleEventIds`, `deleteByUser`.
- **TaskRepository** — `create`, `findById`, `listByUser({ status? })`, `update`,
  `delete`.
- **HabitRepository** — `create`, `findById`, `listByUser`, `update`, `delete`.
- **ScheduledBlockRepository** — `create`, `listByUserInRange`, `setPinned`,
  `delete`, `deleteByUser`.

## Mappers (pure)

In scope for M2 (horizon-independent):
- `toFixedEvent(CalendarEvent) → FixedEvent` — `start/end = startsAt/endsAt.getTime()`.
- `toFlexibleTask(Task) → FlexibleTask` — `dueBy = dueBy.getTime()`.
- `toScheduledBlock(ScheduledBlock) → ScheduledBlock` — derives `sourceType`
  (`task`/`habit`) and `sourceId` from the set FK.

Deferred to M3: `Habit → engine Habit` (needs horizon/timezone to expand
`periods` + `preferredWindows`), working-hours → `workingWindows`, and full
`ScheduleInput` assembly.

## Configuration

- `DATABASE_URL` (dev) and `TEST_DATABASE_URL` (tests), accessed via a small
  typed `config.ts`. No secrets committed; `.env*` is gitignored.

## Error Handling

- Translate Prisma error codes into typed domain errors: `P2002` →
  `ConflictError`, `P2025` → `NotFoundError`; others rethrown unchanged.
- `find*` returns `null` when absent.
- `update` / `delete` on a missing or wrong-user row throws `NotFoundError`.
- Connection failures bubble up; retry/backoff is the caller's concern (M3/M4).

## Testing

- **Mappers** — pure unit tests, no DB, deterministic.
- **Repositories** — integration tests against a real test Postgres
  (`TEST_DATABASE_URL`). A Vitest global setup runs `prisma migrate deploy`
  against the test DB once; each test truncates all tables first for isolation.
  Coverage: CRUD round-trips; `userId` scoping (user A cannot read/mutate user
  B's rows); unique-constraint behavior; the ScheduledBlock one-FK check
  constraint; cascade deletes from `User`.
- One-time setup documented: `createdb notreclaim_test` and a `.env.test` with
  `TEST_DATABASE_URL`.
