# NotReclaim Review 11 ‚Äî design (real jump fix, shell prune, drawer click-outside, planner task panel)

**Date:** 2026-06-13. Branch `feat/review11-planner-panel`. All web-side.

1. **Planner release-jump ‚Äî REAL fix.** Review 10's held-preview + `useLayoutEffect`-clear still jumped in the browser: clearing `held` restored `transition-[top,height]` on the very render that moved `top`, so the browser animated `top` old‚Üínew while `transform` snapped to 0 (jump-back-then-glide). Root: the dragged block must never *paint* a frame where the new `top` is applied while the replan transition is active. Fix in `InteractiveBlock`: a `landing` state. The clear-on-`[startMs,endMs]` layout effect, **only when it was clearing a real drag** (`heldActiveRef`), sets `landing=true` and a double-`requestAnimationFrame` resets it to false after the landed paint. Transition class = `transition-none` while `held || landing`, else `transition-[top,height] duration-300`. React 18 flushes the layout-effect state updates pre-paint, so the first painted frame after commit has the new `top`, `transform:0`, and `transition-none` ‚Üí they cancel, no animation queued. Replans (props change with no prior drag) keep the glide because `landing` is gated on `heldActiveRef`.
2. **Sidebar collapse/hide removed.** The header collapse arrow + the whole hide/pin/overlay/hamburger mechanism are gone ‚Äî the sidebar is always visible. `Sidebar` takes no props; `AppShell` drops pinned/overlay state + the width-animated rail; `TopBar` drops the hamburger (`onShowSidebar`/`sidebarPinned`). `AppShell.testhelper` deleted.

4. **Drawers close on outside click.** New `useClickOutside(ref, onCancel)` hook (mousedown-outside; the opening click already passed before the listener subscribes) wired into `TaskDrawer` and `HabitDrawer`.
